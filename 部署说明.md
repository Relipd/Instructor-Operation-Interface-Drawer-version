# 指导员处理操作平台（横向抽屉版）- 部署说明

## 文件结构
```
指导员处理操作平台-横向抽屉版/
├── manifest.json        # 插件清单（飞书开发者后台导入用）
├── dist/                # 构建产物（可直接上传）
│   └── index.html       # 单文件应用（包含所有 CSS 和 JS）
```

## 设计特点

### 横向抽屉式布局
- **横向卡片列表**：任务以水平滚动的卡片形式展示，信息一目了然
- **右侧抽屉详情**：点击卡片后从右侧滑出详情面板，操作流畅
- **简洁现代 UI**：采用飞书设计语言，视觉清爽专业

### 功能特性
- **任务卡片**：展示任务标题、风险等级、处理人、地点等关键信息
- **状态筛选**：全部 / 待处理 / 待反馈 标签页切换
- **处理人筛选**：下拉选择特定处理人
- **统计面板**：实时显示各状态任务数量
- **确认核查**：待处理任务一键转为待反馈
- **提交反馈**：填写整改措施后标记为已解决
- **风险标签**：R3 红色（高风险）/ R2 橙色（中风险）/ R1 绿色（低风险）

### 多语言支持
- 简体中文 (zh)
- English (en)
- 日本語 (ja)

## 部署步骤

### 方式一：飞书开发者后台（推荐）
1. 登录飞书开发者后台 → 创建企业自建应用 → 添加「多维表格插件」能力
2. 在「多维表格插件」配置页：
   - **插件名称**：指导员处理操作平台
   - **插件入口**：选择「上传 HTML 文件」，上传 `dist/index.html`
3. 配置权限：勾选「查看和编辑多维表格数据」权限（需要写入权限以更新记录状态）
4. 发布版本 → 审核通过后即可在多维表格中使用

### 方式二：本地开发测试
直接用浏览器打开 `dist/index.html` 即可预览效果。

## 界面预览

### 主界面
- 顶部：标题栏 + 筛选标签 + 处理人下拉
- 统计栏：待处理 / 待反馈 / 已解决 数量统计
- 主体：横向滚动的任务卡片列表

### 任务卡片
- 标题（最多两行）
- 风险等级标签
- 处理人信息
- 地点信息
- 状态标签 + 日期

### 抽屉详情
- 任务信息（风险等级、处理人、上报人、时间、地点、状态）
- 问题描述
- 操作按钮（确认核查 / 提交反馈）

## 数据对接

当前为演示模式，使用模拟数据。如需对接飞书多维表格数据，请：

1. 引入飞书 JS SDK：
```html
<script src="https://lf1-cdn-tos.bytegoofy.com/goofy/lark/op/h5-js-sdk-1.5.3/h5-js-sdk-lark.js"></script>
```

2. 在 `loadTasks()` 函数中调用飞书 API 获取数据
